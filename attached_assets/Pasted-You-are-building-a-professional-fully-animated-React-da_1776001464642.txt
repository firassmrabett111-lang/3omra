SECTIONS AND THEIR SPECIFIC CONTENT
Section 1 — Réseau & Architecture: Render the 4 TCP/IP layers as stacked full-width animated cards. Each layer shows its name (spaced uppercase), protocols, a one-sentence description from the PDF, and a latency badge. When hovered, the card expands to show more detail. Below this, render the MODBUS communication architecture diagram showing User Application → Communication Layer (Client + Server) → TCP Management → TCP/IP Stack, with animated arrows showing data flow direction.
Section 2 — Analyse de Trame: This is the centerpiece of the application and must be the most visually impressive section. Show three rows of animated frame fields — one for REQUÊTE (blue), one for RÉPONSE (teal), one for EXCEPTION (coral). Each field (Transaction ID, Protocol ID, Length, Unit ID, Function Code, Data, etc.) is rendered as an individual card with the field name above, the hex value in mono font in the center, and the byte count below. The user can click a "Capturer" button that triggers a typewriter animation where each byte value appears sequentially with a cursor blinking effect. Below the frames, show a detailed explanation panel that decodes the currently selected frame field when clicked — this explanation must come verbatim from the PDF.
Section 3 — Function Codes: Render a premium interactive table with FC01 through FC10+. Each row has a color-coded access badge (READ in teal, WRITE in amber), the hex code in mono font, the function name, and the object type. Clicking a row expands an inline panel showing the exact data format for that function code's request and response frames.
Section 4 — En-tête MBAP: Show each of the 4 MBAP fields as a large interactive card. Each card has the hex value as a large monospace number at the top, the field name, the byte size, and two rows — CLIENT behavior and SERVER behavior — sourced directly from the specification. Animate the cards with a subtle pulse on the accent border.
Section 5 — Paramétrage TCP/IP: Four large cards for TCP_NODELAY, SO_KEEPALIVE, SO_REUSEADDR, and SO_RCVBUF. Each card has a pill showing the recommended setting (ON/OFF or a numeric value), an icon, and a detailed explanation from the PDF. Add a visual diagram showing the Keep Alive timer mechanism as an animated timeline.
Section 6 — Dashboard Live: This section shows the Recharts visualizations. The Line chart shows simulated live MODBUS traffic over the last 12 hours — it must update every 3 seconds with a new data point and animate the line smoothly. The Donut chart shows the distribution of Function Codes used (FC03 dominating at ~42%). The Bar chart shows latency per TCP/IP layer. The horizontal Bar chart shows exception frequency by type. All charts must use the application's color palette, have custom tooltips matching the dark theme, and animated entries.
Section 7 — RTU vs TCP/IP: A split comparison view showing the RTU frame structure on the left (Slave Address + FC + Data + CRC-16) and the TCP/IP ADU on the right (MBAP + FC + Data), rendered as interactive field blocks. Below this, a styled comparison table covering medium, speed, error control, addressing, access mode, simultaneous transactions, and dedicated port.
Section 8 — Séquences: Two side-by-side sequence diagrams rendered in SVG or Canvas — the Client sequence (6 steps from User App to Network and back) and the Server sequence (6 steps from Network to User App and back), with animated message arrows that draw themselves when the section enters the viewport.
